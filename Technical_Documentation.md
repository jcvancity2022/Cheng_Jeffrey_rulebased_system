# Technical Documentation (Assignment 2)

## System Architecture

This project implements a rule‑based generative art system using L‑systems (grammar + expansion) and a Matplotlib‑based architectural renderer. The notebook is organized into three core layers:

1. **Grammar Definitions**: `LSystemGrammar` objects define axiom, rules, and angle for multiple grammars (`tree_grammar`, `koch_grammar`, `desert_grammar`).
2. **String Expansion**: `generate_lsystem_string()` iteratively expands an axiom into a full L‑system string.
3. **Rendering Pipeline**:
   - **L‑system preview (optional)**: `draw_lsystem_matplotlib()` interprets an L‑system string into plotted line segments.
   - **Architectural renderer**: `render_scene()` constructs the resort‑above / mall‑below scene using deterministic geometric rules and palette‑based styling.

## Rules/Grammars Used

- **Tree Grammar** (dense fractal vegetation): `F → FF-[-F+F+F]+[+F-F-F]`, angle 22.5°
- **Koch Grammar** (architectural snowflake): `F → F+F--F+F`, angle 60°
- **Desert Grammar** (sparse resort rhythm): `F → F[+F]F[-F]F`, angle 30°

These grammars demonstrate multiple rule sets and parameter differences (angles, rules, iterations). The final PNG outputs are generated by the architectural renderer rather than direct L‑system drawing.

## Generative Process

1. Define one or more grammars.
2. Expand the grammar into an L‑system string via iterations.
3. Use rule‑based geometry to render the architectural scene:
   - **Resort**: bay widths, floor heights, and wing folds are calculated deterministically.
   - **Mall**: slab depths and facade patterns are generated from parametric iterations.
   - **Perspective**: `perspective_x()` applies courtyard‑style convergence.

## Design Decisions

- **Deterministic structure**: avoids randomness to preserve architectural legibility.
- **Courtyard perspective**: emphasizes depth without collapsing to a single vanishing point.
- **Palette control**: enables clear separation of resort/mall/landscape elements.

## Challenges and Solutions

- **Headless rendering**: `turtle` can fail without a GUI. The notebook uses Matplotlib for reliable rendering and saves PNG outputs.
- **Variation differentiation**: each variation overrides palette and facade/rock patterns to ensure distinct outputs.
